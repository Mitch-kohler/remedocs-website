Features

Everything your PDFs need to be accessible

RemeDocs doesn't just flag issues — it fixes them. Every feature targets a specific accessibility requirement so nothing slips through.

Full tag structure

Builds a complete, standards-compliant tag tree from scratch — paragraphs, headings, lists, figures, and artifacts — no manual tagging needed.

AI alt text

Automatically generates descriptive, context-aware alt text for images, charts, and figures using computer vision — no more empty alt attributes.

Heading hierarchy

Detects and corrects heading levels (H1–H6) to create a logical document outline that screen readers can navigate efficiently.

Table structure

Identifies table cells, row and column headers, and spanning cells, then applies proper TH/TD tags and scope attributes for full data table accessibility.

Link remediation

Converts raw URLs into proper link annotations with meaningful link text, ensuring every hyperlink is keyboard-accessible and clearly described.

Metadata & language

Sets the document title, language attribute, and bookmarks automatically so every PDF is properly identified and navigable.

Compliance

Built for the standards that matter

RemeDocs output is validated against the three major accessibility standards. Every remediated PDF meets all three simultaneously.

WCAG 2.1 AA

Web Content Accessibility Guidelines

The global standard for digital accessibility. RemeDocs ensures your PDFs satisfy all applicable Level AA success criteria for non-web documents.

PDF/UA-1

ISO 14289-1

The dedicated PDF accessibility standard. Every tag, role mapping, and structural element in the output conforms to PDF/UA-1 requirements.

Section 508

U.S. Federal Accessibility Law

Required for U.S. government agencies and their contractors. RemeDocs produces 508-compliant output with optional compliance reports for audit readiness.
